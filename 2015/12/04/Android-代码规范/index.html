Android-代码规范

发表于 2015-12-04   |   分类于 规范   |  
  • 开始之前请详细阅读并遵守Android开发者代码风格指南
  • 统一采用UTF-8 编码
  • 不要使用拼音命名
  • 名称应简洁而富于描述，使用完整单词，避免使用缩写除非该缩写被更广泛使用，例如URL、HTML）
  • 代码中不要出现中文标点字符，例如：‘，’、‘：’
  • 注释中可以书写中文标点字符，便于阅读
  • 注释遵循英文写作习惯，英文标点符号后空一格，避免句子紧凑
  • 颜色色值的字母必须全部大写
  • 布局文件中不要出现硬代码，无法国家化
  • 字体大小使用sp ,其他的使用 dp.
  • 日志输出统一使用自定义logcat
  • 所有Activity 统一继承自定义 BaseActivity
  • 原生图片放在mipmap里面，而drawable主要放一些自定义形状和按钮变色之类的xml.
  • 程序块采用缩进风格编写 , 缩进的空格数为4个。
  • 颜色统一于colors中定义 , 避免直接引用系统颜色(transparent除外)。
  • 尽可能使用静态内部类而不是非静态内部类。每一个非静态内部类实例都会持有一个外部类的引用，若该引用是 Activity 的引用，那么该 Activity 在被销毁时将无法被回收。如果你的静态内部类需要一个相关 Activity 的引用以确保功能能够正常运行，那么你得确保你在对象中使用的是一个 Activity 的弱引用，否则你的 Activity 将会发生意外的内存泄漏。

一、命名规则

1. 包命名

包名由小写的字母组成，默认以 com.ecdj 开头，然后接上根据功能划分的模块名。

com.ecdj.ui
com.ecdj.widget

2. 类和接口命名

名称的首字母需要大写，如果由多个单词组成，那么每个单词的首字母需要大写，其他字母小写。

activity , 命名以Activity为后缀 , 如: class LoginActivity
fragment , 命名以Fragment为后缀 , 如: ShareDialogFragment
工具类 , 命名以Util为后缀
interface OnClickListener

3. 方法的命名

采用驼峰命名法来命名。方法名应该是动词或者是对功能的一个描述 , 第一个单词的首字母小写 , 其它单词的首字母大写

初始化方法 , 命名以init开头 , initView
按钮点击方法 , 命名以to开头 , toLogin
通过异步加载数据的方法 , 命名以load开头 , loadData
布尔型的判断方法 , 命名以is , has或者具有逻辑意义的单词(check,equals)开头 , isEmpty

4. 变量的命名规则

Non-public, non-static field names start with m.
非公共的、非静态的域变量用m前缀.
Static field names start with s.
静态域变量用s前缀.
Other fields start with a lower case letter.

例如:

public class MyClass {
    // 常量需要声明为final static形式，组成名称的单词必须全部大写，单词之间用下划线隔开。
    public static final int SOME_CONSTANT = 42;  // 常量

    public int publicField;                // 公有的成员变量

    private static MyClass sSingleton; // 静态成员变量

    int mPackagePrivate;                // 非静态成员变量
    private int mPrivate;
    protected int mProtected;
}

5. 异常的命名

名称必须以Exception结尾。

NullPointerException

6. 布局的命名 xml的命名

name是 模块的名称+功能的描述 ，单词必须全部小写，单词之间用下划线隔开。

组件类型{范围|模块}功能，范围可选，只在有明确定义的范围内才需要加上

类型 模板 事例
Activity activity_module_name.xml activity_home_main.xml
Tab tab_module_name.xml tab_home_menu.xml
Dialog dialog_module_name.xml dialog_home_login.xml
ListItem item_module_name.xml tab_home_products.xml
Dialog dialog_module_name.xml item_home_contact.xml
ListListView的HeaderView header_list_xxxx.xml
ListListView的FooterView footer_list_xxxx.xml

7. 控件ID的命名

组成名称的单词必须全部小写，单词之间用下划线隔开，名称不需要复杂的层级定位，只需要准确描述所代表控件的功能作用即可，通常我们在名称前使用控件类型的缩写前缀来避免重复起名的麻烦。例如：控件类型+{模块名|范围}+意义

name是模块的名称或功能的描述。

类型 模板 事例
TextView tv_name tv_title
EditText et_name et_mail
RelativeLayout rlyt_name rlyt_login
LinearLayout llyt_name llyt_login

控件类型缩写对照表
Layout 统一所写为： lyt

控件类型 源文件中缩写 资源文件中缩写
View V v

ImageButton Ibtn ibtn
RadioButton Rbtn rbtn
ToggleButton Tbtn tbtn
CheckBox Chk chk

ProgressBar Pbar pbar

RelativeLayout Rlyt rlyt
LinearLayout Llyt llyt
TableLayout Tlyt tlyt
FrameLayout Flyt flyt

8. 图片的命名

name是模块的名称或功能的描述。

前缀_[功能描述|范围|后缀]，控件、范围、后缀可选，但功能描述和范围至少要有一个。

前缀 描述 示例
ic_ 图标 ic_launcher.png
bg_ 背景图片 bg_title.png 
图标类，添加ic前缀
背景类，添加bg前缀
分隔类，添加div前缀

区分状态时，默认状态，添加normal后缀
区分状态时，按下时的状态，添加pressed后缀
区分状态时，选中时的状态，添加selected后缀
区分状态时，不可用时的状态，添加disable后缀

多种状态的，添加selector后缀（一般为ListView的selector或按钮的selector）

9. 其他资源命名

遵循见名知意的原则，组成名称的单词必须全部小写，单词之间用下划线隔开。

<string name="welcome">欢迎</string>
<color name="white">\#FFFFFF</color>

二、注释

养成良好的注释习惯，对提升自己的编程能力和团队合作能力有很大的益处。一般情况下，源程序有效注释量必须在20％以上。

说明：注释的原则是有助于对程序的阅读理解，在该加的地方都加了，注释不宜太多也不能太少，注释语言必须准确、易懂、简洁。

1. 文件注释

源文件的开头需要注释说明当前的文件信息，包括文件名、作者、版本信息、日期等。

/** 
* Copyright (C) 2007-2015, by HuaShengRun, Shenzhen, All rights reserved.
* 
*
* File: RequestManager.java 作用及实现的功能描述
* Author: JiongBull
* Version: 1.0
* Create: 2015-8-10
*/

2. 类注释

类和接口统一添加javadoc , 格式如下

/**
* [类或接口的描述信息]
*
* @author ${USER}
* @date ${DATE}
*/

3. 方法注释

下面几种方法，都必须添加javadoc注释，说明该方法的用途和参数说明，以及返回值、异常信息等。

1.接口中定义的所有方法
2.抽象类中自定义的抽象方法
3.抽象父类的自定义公用方法
4.工具类的公用方法

/**
* 输出日志.
*
* @param context Context
* @param logLevel 日志等级
* @param tag 标签
* @param content 输出内容
* @param throwable 异常信息, 若没有异常可为空
*/
private static void trace(final Context context, final int logLevel, 
                                                final String tag, 
                                                final String content, 
                                                final Throwable throwable) {

...

}

4. 域变量和常量注释

下面几种情况下的常量和变量，都要添加注释说明，优先采用在上方添加注释。

1.接口中定义的所有常量
2.公有类的公有常量
3.枚举类定义的所有枚举常量
4.实体类的所有属性变量

/**
* 现金券
**/
public static final int TYPE_CASH = 1;

/**
* 抵扣券
**/
public static final int TYPE_DEBIT = 2;

/**
* 抵扣券
**/  
public static final int TYPE_DISCOUNT = 3

5. 其他注释

单行注释

view.setImageResource(defaultImageResId); // 无图片显示默认图片


区块注释


/* 网络图片添加渐变动画 */


Resources resource = mContext.getResources();

final TransitionDrawable transitionDrawable =
                new TransitionDrawable(new Drawable[] {
                    new ColorDrawable(android.R.color.transparent),
                    new BitmapDrawable(resource, bmpResponse)   });

view.setBackgroundResource(defaultImageResId);
view.setImageDrawable(transitionDrawable);
transitionDrawable.startTransition(RequestManager.FADE\_IN\_TIME);

6. XML注释

<!-- Base application theme. -->
<style name="AppTheme" parent="android:Theme.Holo.Light.NoActionBar">
<!-- Customize your theme here. -->
</style>

约定俗成的一些注释原则

1. get / set 方法的注释

通过IDE工具批量生成域变量的 get / set 方法可以不用注释。

2. 第三方库生成代码的注释

第三方库生成的代码可以不用注释，例如greenDao生成的代码。

3. 通用类的注释

通用类的公共方法应全部添加注释。

4. 一般情况下的注释

例如在activity、service等文件中，只需对关键的类变量进行注释，方法建议全部添加注释。

三、书写，排版

1. 前大括号不单独开行

public void method()
{    //bad
}  

public void method(){    //good
}

2. 空行加入原则

两个方法之间;
方法内的两个逻辑段之间;
方法内的局部变量和方法的第一条逻辑语句之间;
常量和变量之间;
相对独立的程序块之间必须加空行;

示例: 如下例子不符合规范

if(!validate(ni)){...//code
}
repCode=ssnData[index].rep_code;
repCount=ssnData[index].rep_count;

应如下书写

if(!validate(ni)){ 
...//code
}

repCode=ssnData[index].rep_code;
repCount=ssnData[index].rep_count;

3.一行声明一个变量，不要一行声明多个变量，这样有利于写注释。

/**
* 参数1
*/
private String param1; 

/**
* 参数2
*/
private String param2;

4.使用快捷键进行代码自动格式化。

Windows：CTRL+ALT+L

Mac：OPTION+COMMAND+L

5. 较长的语句(100字符)要分成多行书写 , 长表达式要在低优先级操作符处划分新行 , 操作符放在新行之首 , 划分出的新行要进行适当的缩进 , 使排版整齐

示例 :
permCountMsg=NO7_TO_STAT_PERM_COUNT_LEN
　　　　　　　　+STAT_SIZE_PER_FRAM*sizeOf(UL);
actTaskTab[frame_id * STAT_TASK_CHECK_NUMBER + index].occupied
　　　　= statPoi[index].occupied;

4. if、for、do、while、case、switch、default等语句自占一行，且if、for、do、while等语句的执行语句部分无论多少都要加括号{}。

示例 : 如下例子不规范
if(userCr==null) return;
应如下书写
if(userCr==null){
　　return;
}
